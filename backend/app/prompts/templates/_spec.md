## 摘要規範

- 使用繁體中文為主，英文為輔。
- 摘要架構清晰易讀，適當使用表格、列點、流程圖等方式提升易讀性。
- 摘要內容簡要、正確、精闢且易懂，並避免任何冗言贅字。
- 摘要完成後請再次針對內容進行校稿，避免任何錯誤與幻覺。

---

## Markdown 格式規範

為確保摘要被正確渲染，請務必遵守以下規則：

### 段落分隔
- **段落之間必須使用空行分隔**
- 連續的文字行會被視為同一段落

### 列表格式
- 列表項目可以連續撰寫
- **列表結束後必須加空行** 才能開始新段落

### 標題格式
- 標題前後都應有空行
- 使用 `#`, `##`, `###` 表示不同層級

### 範例

✅ **正確**：
```markdown
這是第一段。

這是第二段。

## 標題

- 列表項目 1
- 列表項目 2

下一段落開始。
```

❌ **錯誤**（會被合併）：
```markdown
這是第一段。
這會和上一行合併。
- 列表
下一段也會出問題。
```
