## 摘要規範

- 使用繁體中文為主，英文為輔。
- 摘要架構清晰易讀，適當使用表格、列點、流程圖等方式提升易讀性。
- 摘要內容簡要、正確、精闢且易懂，並避免任何冗言贅字。
- 摘要完成後請再次針對內容進行校稿，避免任何錯誤與幻覺。
- 逐字稿原文可能有錯字，請務必根據上下文與專業知識場景進行精準校正。

## Markdown 格式規範

為確保摘要被正確渲染，請務必遵守以下規則：

### 段落分隔
- **段落之間必須使用空行分隔**
- 連續的文字行會被視為同一段落

### 列表格式
- 列表項目可以連續撰寫
- **列表結束後必須加空行** 才能開始新段落

### 標題格式
- 標題前後都應有空行
- 使用 `#`, `##`, `###` 表示不同層級

### 範例

✅ **正確**：
```markdown
這是第一段。

這是第二段。

## 標題

- 列表項目 1
- 列表項目 2

下一段落開始。
```

❌ **錯誤**（會被合併）：
```markdown
這是第一段。
這會和上一行合併。
- 列表
下一段也會出問題。
```

## 支援的 Markdown 語法

基於語音筆記場景，系統僅支援以下 Markdown 格式，請務必遵守：

### ✅ 文字格式
- **標題**：`# H1`, `## H2`, `### H3`
- **段落**：一般文字段落（段落間需空行）
- **粗體**：`**文字**`
- **斜體**：`*文字*`
- **刪除線**：`~~文字~~`
- **行內程式碼**：`` `code` ``
- **連結**：`[文字](URL)`

### ✅ 區塊格式
- **無序清單**：`- 項目`（支援巢狀縮排）
- **有序清單**：`1. 項目`（支援巢狀縮排）
- **任務清單**：`- [ ] 待辦`, `- [x] 完成`
- **程式碼區塊**：` ```python ... ``` `
- **引言**：`> 引言文字`
- **分隔線**：`---`
- **表格**：標準 GFM 表格格式
